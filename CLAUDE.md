# Git ブランチ戦略

このプロジェクトは Git Flow を採用しています。

## ブランチ構成

```
        V1.0.0        V2.0.0                    V2.0.0
main   ----●------------>●-------------------------------->●- - -
            \           ↑ ↑6                               ↑
             \          |                                  |
hotfix         \      ●-+                                  |
                \     ↑                                    |
                 \    |                                    |
release           \   |                          ●-------->●
                   \  |                         ↑4         |5
                    \ |                    3   /            \
develop   ●-->●-->●--\-->●---------->●-------->●------------>●
               ↑1    \ ↑1          ↑2         \
               |      \|          /            \
feature        |       ●-------->●              \
               |
feature        ●-->●-->●-->●
```

## 各ブランチの役割

| ブランチ | 役割 |
|----------|------|
| `main` | リリース済みコード。タグ（V1.0.0, V2.0.0 等）で管理 |
| `develop` | 開発の統合ブランチ。次のリリース向けの変更を集約 |
| `feature/*` | 機能開発用。`develop` から分岐し、`develop` にマージ |
| `release/*` | リリース準備用。`develop` から分岐し、`main` と `develop` にマージ |
| `hotfix/*` | 緊急バグ修正用。`main` から分岐し、`main` と `develop` にマージ |

## 運用ルール

- `feature` → `develop` へマージ（PR必須）
- `develop` → `release` へ分岐してリリース準備
- `release` → `main` と `develop` の両方にマージ
- `hotfix` → `main` から分岐し、`main` と `develop` の両方にマージ
- `main` への直接プッシュは禁止
