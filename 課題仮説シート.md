# 課題仮説シート

## プロダクト名

GitGUI

## 一言で言うと？



## 1. 誰の課題か？（ターゲットユーザー）

- **どんな人？**：未経験からエンジニアになり、Gitというもの自体がわからない人
- **人数の規模感**：1万人(毎年のエンジニア志望人数の1~2割程度)
- **具体的に1人思い浮かべてください**：インターン開始時の私

## 2. どんな課題か？

- **困っていること**：CLIを実行するだけでは何をしているかが把握できず、実態がつかめない。
- **今どうやって対処しているか**：さまざまな方の残した備忘録を見漁って学習。
- **なぜそれでは不十分か**：圧倒的に時間が掛かる上、認識が合っているかが不明瞭。

## 3. なぜ今この課題を解くのか？

エンジニアとしてGitの動作実態を掴む必要があるため

## 4. どう解決するか？

- **解決アプローチ（1-2文）**：git add/commit/push等のコマンドに同期して画像内のファイルブロックが動き、何が行われたかの説明文が表示される。
次の実行フェーズの提案や現状のエラーに対する解決案を提示する。
- **モックで見せるコア機能**：gitコマンドを打つことでにファイルの動きが確認できる。
- **ユーザーがこのプロダクトを使ったら何が変わるか**：視覚的にGitコマンドによる動作の内容がわかりやすくなり、学習時間が削減される。
